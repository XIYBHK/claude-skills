# 更新日志生成器

这个 skill 能将技术性的 git 提交记录转换为精美、用户友好的更新日志，让你的客户和用户真正理解和欣赏产品的变化。

## 使用场景

- 准备新版本的发布说明
- 创建每周或每月的产品更新摘要
- 为客户编写变更文档
- 编写应用商店提交的更新日志
- 生成更新通知内容
- 创建内部发布文档
- 维护公开的更新日志/产品更新页面

## 功能说明

1. **扫描 Git 历史**：分析特定时间段或版本之间的提交记录
2. **分类变更**：将提交归类为逻辑类别（新功能、改进、bug修复、破坏性变更、安全更新）
3. **技术语言 → 用户语言**：将开发者提交转换为客户能理解的描述
4. **专业格式化**：创建清晰、结构化的更新日志条目
5. **过滤噪音**：排除内部提交（重构、测试等）
6. **遵循最佳实践**：应用更新日志指南和你的品牌语调

## 使用方法

### 基础用法

在项目仓库中使用：

```
生成自上次发布以来的更新日志
```

```
生成过去一周的所有提交的更新日志
```

```
创建 2.5.0 版本的发布说明
```

### 指定日期范围

```
创建 3月1日 到 3月15日 之间所有提交的更新日志
```

### 使用自定义指南

```
生成自 v2.4.0 以来的更新日志，使用 CHANGELOG_STYLE.md 中的格式指南
```

## 示例

**用户**："生成过去7天的更新日志"

**输出**：
```markdown
# 更新 - 2024年3月10日这周

## ✨ 新功能

- **团队工作区**：为不同项目创建独立的工作区。邀请团队成员，
  让一切井井有条。

- **键盘快捷键**：按 ? 查看所有可用快捷键。
  无需触摸鼠标即可快速导航。

## 🔧 改进

- **更快的同步**：文件现在在设备间的同步速度提升2倍
- **更好的搜索**：搜索现在包含文件内容，不仅仅是标题

## 🐛 修复

- 修复了大图片无法上传的问题
- 解决了定时发布中的时区混淆问题
- 修正了通知徽章计数错误
```

**灵感来源**：Manik Aggarwal 在 Lenny's Newsletter 中的使用案例

## 使用建议

- 在 git 仓库根目录运行
- 指定日期范围以生成聚焦的更新日志
- 使用 CHANGELOG_STYLE.md 保持格式一致
- 发布前审查和调整生成的更新日志
- 直接保存输出到 CHANGELOG.md
